# agent-memory

CLI-агент для взаимодействия с LLM с сохранением контекста.

---

## Описание

`agent-memory` — это Python CLI-инструмент для интерактивного общения с LLM.  
Агент сохраняет историю диалога в SQLite и загружает её при следующем запуске.

## Возможности

- Интерактивный чат с LLM
- Сохранение контекста диалога между сессиями
- Подсчет токенов и стоимости
- Команды управления (help, reset, history)
- Запрос на загрузку истории при запуске

## Установка

```bash
cd agent-memory
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

## Настройка

Создайте файл `.env`:

```bash
cp .env.example .env
```

Отредактируйте `.env`:

```env
OPENAI_API_KEY=your_api_key_here
OPENAI_BASE_URL=https://api.openai.com/v1
OPENAI_MODEL=openai/gpt-4o-mini
```

## Использование

Запуск чата:

```bash
python -m app.main
```

При первом запуске начнется новый диалог.  
При последующих запусках программа предложит загрузить сохраненную историю.

### Команды чата

| Команда | Описание |
|---------|----------|
| `help` | Показать справку |
| `stats` | Показать статистику запросов |
| `reset` | Очистить историю разговора |
| `history` | Показать историю сообщений |
| `exit` | Выйти из чата |

### Пример сессии

```
Agent Memory CLI — Чат с LLM с памятью
Введите 'help' для списка команд

Команды: help, stats, reset, history, exit

Найдена сохраненная история (4 сообщений).
Загрузить историю? (y/n): y
История загружена.

Вы: Привет! О чем мы говорили?
[Думаю...]

Агент: Мы говорили о том, что ты спросил кто я...

[Думаю...]

Агент: В начале разговора ты спросил "Привет! Кто ты?"

Вы: exit
История сохранена.
```

## Метрики

Программа подсчитывает:

- **Input tokens** — токены в запросе (включая историю)
- **Output tokens** — токены в ответе
- **Total tokens** — суммарное количество
- **Cost** — примерная стоимость

## Как это работает

1. При каждом запросе к агенту история сохраняется в SQLite
2. При выходе (`exit` или Ctrl+C) история автоматически сохраняется
3. При запуске проверяется наличие сохраненной истории
4. Если история есть — пользователю предлагается загрузить её
5. При загрузке агент продолжает диалог с учетом предыдущих сообщений
