# agent-managed-context

CLI-агент для взаимодействия с LLM с управлением контекстом.

---

## Описание

`agent-managed-context` — Python CLI-инструмент для интерактивного общения с LLM с поддержкой 3 стратегий управления контекстом.

## Возможности

- **3 стратегии управления контекстом:**
  - Sliding Window — хранит последние N сообщений
  - Sticky Facts — key-value память + последние N сообщений  
  - Branching — ветвление диалога с переключением между ветками

- Демо режим для автоматического тестирования
- Сравнение стратегий по памяти, качеству, токенам
- Сохранение контекста между сессиями
- Подсчет токенов и стоимости

## Использование

Запуск чата:

```bash
python -m app.main
```

### Команды чата

| Команда | Описание |
|---------|----------|
| `help` | Показать справку |
| `stats` | Показать статистику и сравнение стратегий |
| `reset` | Очистить историю разговора |
| `history` | Показать историю сообщений |
| `strategy` | Показать/изменить стратегию |
| `switch` | Переключить ветку (Branching) |
| `facts` | Показать факты (Sticky Facts) |
| `branches` | Показать ветки (Branching) |
| `demo` | Запустить демо режим |
| `exit` | Выйти из чата |

---

## Демо режим

Команда `/demo` запускает автоматическое тестирование выбранной стратегии:

```bash
demo
# Выберите стратегию: 1 - Sliding, 2 - Sticky, 3 - Branching
```

Демо показывает:
- Все сообщения и ответы
- Статистику по токенам
- Анализ памяти (что LLM запомнил/забыл)

После нескольких запусков `/stats` покажет сравнение всех стратегий.

---

## Стратегии управления контекстом

### 1. Sliding Window

- Хранит только последние N сообщений (по умолчанию N=5)
- Всё остальное отбрасывается

### 2. Sticky Facts

- Блок "facts" (ключ-значение)
- Автоматически извлекает факты из сообщений
- Факты + последние N сообщений в запросе

### 3. Branching

- 2 ветки: main и alt
- Переключение командой `/switch`
- Каждая ветка хранит полный контекст

## Тестовые сценарии

### Сценарий 1: Проверка памяти (10 сообщений)

Для Sliding Window и Sticky Facts:

```
1. Запомни: моего кота зовут Барсик
2. Я живу в городе Москва
3. Мне 25 лет
4. Я работаю врачом
5. Сколько будет 2+2?
6. Какой сегодня день недели?
7. Расскажи анекдот
8. Что такое Python?
9. О чем был последний вопрос?
10. Как зовут моего кота и где я живу?
```

**Что проверяет:**
- Вопрос 9 (в контексте) — должен помнить
- Вопрос 10 (за пределами N=5) — зависит от стратегии

### Сценарий 2: Branching (17 сообщений)

```
main: Делаем интернет-магазин одежды
main: Бюджет 100 тысяч, срок 2 месяца
main: Нужна корзина и оплата картой
/switch
alt: А если добавить доставку дронами?
alt: Это легально в России?
alt: Сколько стоит?
/switch
main: Продолжим про магазин, добавим акции
main: Про сроки доставки забыли
/switch
alt: Сколько стоит внедрение дронов?
alt: Какие ограничения по весу?
alt: Реально сделать за 2 месяца?
/switch
main: Подведи итог по магазину
main: Подведи итог по доставке дронами
```

## Сравнение стратегий

| Стратегия | Плюсы | Минусы | Качество |
|-----------|-------|--------|----------|
| Sliding Window | Простота, автоматически | Теряет ранний контекст | ++ |
| Sticky Facts | Сохраняет факты, прозрачность | Ограниченный набор фактов | +++ |
| Branching | Полный контекст, альтернативы | Сложнее управление | +++ |

## Как это работает

1. Запустите `python -m app.main`
2. Введите `/demo` для тестирования

5. После нескольких запусков введите `/stats` для сравнения
